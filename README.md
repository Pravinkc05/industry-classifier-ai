# Industry Classifier AI — LLM-Powered Business Classification

An automated tool that classifies a company's business description against standard industry codes (NAICS) using NLP and LLM-based semantic matching — the kind of classification underwriters do manually when assigning industry codes to new accounts.

Originally built as part of an automated underwriting pipeline; this repo is a standalone, sanitized rebuild of that architecture, demonstrating the NLP/LLM matching logic independent of any proprietary data sources.

## The problem this solves

Manually mapping a company's stated business activities to standard industry classification codes is slow and inconsistent — different underwriters often pick different codes for similar businesses. This tool automates that judgment call using semantic similarity, while keeping the final decision human-reviewable: it returns ranked candidates with confidence scores, not a single forced answer.

## Architecture

```
Company description → NER extraction → LLM summarization → Embedding model
                                                                    ↓
                              Streamlit results ← Cosine similarity ← Industry code set
```

Standard industry code descriptions (NAICS) are embedded once via `precompute_code_embeddings.py` and cached to disk, so each new company lookup only needs to embed the company's own profile and compare against the cached set.

## Dual-provider design

Both the LLM summarization step and the embedding step support two interchangeable backends, switched with a single config value — no other code changes needed:

| Component | `openai` | `ollama` / `local` |
|---|---|---|
| Summarization | OpenAI chat completions | Local LLM via Ollama |
| Embeddings | OpenAI embeddings API | sentence-transformers (on-device) |

Defaults to `ollama` + `local`, so the app runs fully offline with no API key out of the box. Switch to OpenAI for either step by setting environment variables:
```bash
export LLM_PROVIDER=openai
export EMBEDDING_PROVIDER=openai
export OPENAI_API_KEY=your_key_here
```

## Features

- Named Entity Recognition (spaCy) pulls organization names and industry-relevant terms from raw business descriptions
- LLM summarization condenses noisy/marketing-heavy company text into a clean, comparable business profile
- Embedding-based semantic matching — catches cases where a company describes itself differently than a code's exact wording, unlike keyword search
- Returns top-k ranked matches with similarity scores, so a human reviewer makes the final call
- Streamlit interface for quick manual lookups
- Swap LLM/embedding providers via config, with zero changes to the matching logic

## Setup

```bash
# 1. Install dependencies
pip install -r requirements.txt
python -m spacy download en_core_web_sm

# 2. (Optional) For local-first mode, install Ollama and pull a model:
ollama pull llama3
# This is the default -- no API key needed to get started.

# 3. Precompute embeddings for the industry code reference set (run once,
#    and again any time you edit data/naics_codes.csv)
python precompute_code_embeddings.py

# 4. Launch the app
streamlit run app.py
```

## How it works

1. **NER (`core/ner.py`)** — extracts organization names and industry keywords from the raw description as a cheap pre-filter before the LLM step.
2. **Summarization (`core/summarizer.py`)** — feeds the description (plus the extracted entities as context) to an LLM, asking for a clean, factual business-profile paragraph with marketing language stripped out.
3. **Embedding (`core/embeddings.py`)** — vectorizes that summary, and loads the precomputed industry code embeddings from cache.
4. **Matching (`core/matcher.py`)** — ranks the cached code embeddings by cosine similarity to the company's profile embedding, returning the top-k with scores.
5. **UI (`app.py`)** — Streamlit app that ties the pipeline together and displays ranked results for review.

## What this demonstrates

- Designing a provider-agnostic abstraction layer (LLM and embedding backends are swappable via config, not hardcoded) — a pattern that matters once you've been burned by a single vendor's pricing or rate limits
- Practical use of embeddings for semantic matching rather than keyword search
- A human-in-the-loop design: the model ranks and scores, a person confirms — relevant for any AI tool used in a regulated or judgment-heavy domain
- NER + LLM summarization + embeddings combined into a single working pipeline, not used in isolation
